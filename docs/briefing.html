운영 구조

사용자 경험은 간단하지만, 내부는 출처·정규화·개인화·안전확인을 분리합니다.

1

사용자 요청

모바일, 메신저, 보이스에서 아이·지역·날짜 조건을 입력합니다.

2

스킬 라우팅

식단, 학교, 놀이, 생활, 트렌드 등 맞는 스킬로 연결합니다.

3

공통 정규화

지역코드, 학교코드, 기관 ID, 서비스 ID를 한 곳에서 해석합니다.

4

공식 데이터 우선

NEIS, 알리미, TourAPI, 공공예약, 복지 데이터를 우선 조회합니다.

5

확인 후 행동

캘린더, 리마인더, 신청 링크, 구매 후보는 부모 확인 후 진행합니다.

데이터 원천 우선순위

신뢰도와 법적 안정성을 기준으로 공식 원천을 먼저 쓰고, 민간·스크래핑은 보강층으로 제한합니다.

우선순위 원천 사용 영역 주의점
1 공식 API·공시 NEIS 급식·학사, 유치원/어린이집 알리미, 정부24, 복지로 코드 resolver와 갱신 시점 표시가 필요합니다.
2 공공데이터 파일 놀이시설, 도시공원, 문화축제, 지역 행사 백필 캐시와 정기 동기화가 필요합니다.
3 지도·지역검색 API 좌표 보정, 장소 최신성, 길찾기 힌트 쿼터와 키 관리, 이용 약관을 분리합니다.
4 제한적 스크래핑 백화점·마트 문화센터 공개 강좌 페이지 전체 미러링 금지, 약관·robots·로그인 경계 확인이 필요합니다.
5 커뮤니티·블로그 보조신호 트렌드, 후기 흐름, 큐레이션 보강 공식 정보와 섞지 않고 source type을 분리합니다.

향후 계획

코드 구현 전에는 스펙과 원천 검증을 마치고, 이후 공식 데이터 MVP부터 단계적으로 확장합니다.

M0 스펙·소스 인벤토리

제품 스펙, 데이터 계약, 프로젝트 구조, API 키·라이선스·법적 경계 정리.

M1 공식 데이터 MVP

NEIS resolver, 급식·알레르기, 학교 일정, 유치원·어린이집 비교.

M2 외출·행사 레이어

TourAPI, 놀이시설, 공원, 공공예약, 도서관, 평생교육 추천.

M3 신청 가이던스

정부24·복지로·서울예약의 자격 요약, 필요서류, 마감 리마인더.

M4 문서 OCR·캘린더

가정통신문 사진에서 일정·준비물·마감 추출 후 확인 등록.

M5 민간 소스 확장

문화센터 공개 강좌 감시, 수동 검수, conservative crawler 운영.
