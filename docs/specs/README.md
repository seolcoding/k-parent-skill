# Specs

This directory turns the research notes into implementation-ready planning documents.

These files are intentionally documentation-only. They describe future structure, contracts, and sequencing without changing runtime code.

| Spec | Purpose |
| --- | --- |
| [product-spec.md](product-spec.md) | Product scope, MVP, user flows, non-goals, and quality gates |
| [data-contracts.md](data-contracts.md) | Shared entities, source priority, normalization contracts, and privacy constraints |
| [project-structure.md](project-structure.md) | Future repository structure and milestone plan while preserving `k-skill` conventions |

Research notes live under `docs/research/`. Durable implementation rules should be promoted into `docs/architecture.md`, `docs/productization-gstack.md`, and this spec set.
